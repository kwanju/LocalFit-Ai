# ADR-002: MVP 토폴로지 — 데스크탑 단독 백엔드

- **상태**: Accepted
- **작성일**: 2026-05-21
- **관련 ADR**: ADR-014 (프로세스 라이프사이클)

## 컨텍스트

PRD는 "오프라인 우선"과 헬스장에서의 C2S 모드를 동시에 명시한다. 두 요구는 부분적으로 충돌한다. 사용자(1인)의 환경은 Windows + RTX 5090 데스크탑이며, 폰은 Android다.

사용자는 폰↔데스크탑 동시 사용을 원하지만, MVP 단계에서는 데스크탑 단독으로 시작하고 폰 시나리오는 P1으로 미루기로 했다.

## 결정

### MVP (P0)
- 데스크탑 단독으로 동작한다
- FastAPI 백엔드를 `127.0.0.1:8000`에 바인딩하여 외부 노출을 차단한다
- 같은 데스크탑의 브라우저가 UI 클라이언트다

### P1 진입 시 변경
- 바인딩 주소를 `0.0.0.0`으로 변경 (코드 1줄)
- Tailscale 설치 및 데스크탑·폰 연결
- Wake-on-LAN 스크립트 추가

### 코드 구조
처음부터 "다중 클라이언트 가능"을 가정한다. WebSocket 연결은 N개 허용하되, 세션은 단일 사용자(`UserProfile.id=1`) 가정으로 단순화한다.

### P1 전환 트리거
"MVP 사용 2주 후 회고 시점" 또는 "헬스장 사용 의지가 명확해진 시점" 중 먼저 도달하는 시점. 명시적 회고 없이 코드를 P1으로 끌고 가지 않는다.

## 결과

### 긍정
- MVP에서 인증·HTTPS·터널링 코드 0줄
- P1 진입 시 코드 변경 최소 (인프라 변경이 대부분)
- 보안 부담 최소화

### 부정
- 헬스장 시나리오는 P1 이전엔 동작하지 않음 — 사용자 수용
- "P1 전환 트리거"를 명확히 안 하면 영원히 P0에 머물 위험

## 대안

| 후보 | 탈락 사유 |
|---|---|
| 포트포워딩 + 자체 인증 | 보안 부담 과중, 1인 프로젝트 부적합 |
| Cloudflare Tunnel | 트래픽이 Cloudflare 경유 → "로컬 우선" 정신과 거리감 |
| MVP부터 Tailscale 도입 | P0 범위 확대, 미루는 것이 단순 |
| ngrok 등 터널 서비스 | Tailscale 대비 보안·운영 비용 ↑ |
