# ADR-028: 첫 세션 대화형 체력검증

- **상태**: Proposed (2026-06-10)
- **관련 ADR**: ADR-013 (능동 코치), ADR-024 (플랜 입력), ADR-025 (기준선 저장)

## 컨텍스트

v3 는 신규 사용자의 실제 체력을 모른 채 **고정값**(예: 푸시업 10회 3세트)을 던졌다. 온보딩 폼(`Onboarding.tsx`)에 체력 측정 입력칸(풀업/푸시업/스쿼트/플랭크 최대)이 있고 `/onboarding` 에 저장되지만, 코치 대화로 이를 끌어내 **함께 계획을 세우는 흐름은 없었다**. v3 에서 의도적으로 보류(YAGNI + master 동결)했고 v4 로 이월됐다(v4-vision §6-11, ADR-028).

## 결정

- **첫 세션 = 대화형 체력검증.** 코치가 온보딩 폼 값을 **시드**로 대화로 확인/보정한다.
  - 예: "푸시업 15개로 적으셨는데 맞나요? 그럼 70%인 10개로 가볍게 시작할게요."
- **자가보고 방식** — 실측 1세트 최대치 측정은 하지 않는다(첫 경험 부담 + 부상 위험).
- **보수적 시작**: 자가보고치의 일정 비율(예: 70%)로 출발. 부정확성은 보수적 시작 + 빠른 재조정으로 흡수.
- **결과 저장**: `fitness_baseline`(ADR-025 1층 구조화 필드) → 플랜(ADR-024)의 기준선.
- **재평가 트리거**: N주 경과 또는 사용자의 "너무 쉬움/어려움" 피드백 시 재측정 대화.
- **안 행복한 경로**:
  - 온보딩 폼 미입력 → 대화로 처음부터 수집.
  - 사용자가 수치 답을 회피 → 매우 보수적인 기본값으로 시작 후 빠르게 조정.

## 결과

### 긍정
- 대화형 코칭의 첫 적용 사례 — v4 차별점
- 신규 사용자에게 맞춤 시작 강도(고정값 탈피)
- 보수적 시작으로 부상 위험 낮음
- 기준선이 플랜·컨디션 조절의 공통 입력

### 부정
- 자가보고는 부정확 가능(보수적 시작·재조정으로 완화)
- 대화 흐름 설계·검증 비용(능동 코치 프롬프트 확장)

## 대안

| 후보 | 탈락 사유 |
|---|---|
| 대화형 + 실측 세트 | 첫 세션 "한계까지 테스트" = 부담 + 부상 위험 |
| v3 고정값 유지 | 대화형 코칭이라는 v4 차별점 포기 |
| 폼만으로 자동 산정(대화 없음) | 값 검증·맥락(부상 등) 반영 못 함 — v3 한계 그대로 |
