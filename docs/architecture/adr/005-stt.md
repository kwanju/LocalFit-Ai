# ADR-005: STT — faster-whisper large-v3-turbo + 16kHz 강제

- **상태**: Accepted (2026-05-31)
- **관련 ADR**: ADR-007 (VAD), ADR-011 (Pipecat), ADR-012 (도메인 어댑터)

## 컨텍스트

v1에서 faster-whisper large-v3-turbo가 한국어 STT로 검증됐다. RTX 5090 + CTranslate2 4.7.2 + cu128에서 float16 정상 동작. **단, v1에서 발견된 known_constraint**: faster-whisper는 입력을 16kHz로 가정하므로 32kHz WAV는 transcript가 어긋남. v1은 STT 어댑터에서 리샘플 안 함 → 프론트엔드 책임이었음.

Pipecat은 `pipecat.services.whisper.stt.WhisperSTTService`로 faster-whisper를 공식 지원하지만, **샘플레이트 정합성 보장**은 명시되지 않는다. v3는 어댑터 진입부에서 항상 16kHz로 리샘플하도록 강제한다.

## 결정

- **STT 모델 = faster-whisper `large-v3-turbo`** (한국어 정확도·속도 균형 최적)
- **device**: `cuda`, **compute_type**: `float16`
- **샘플레이트 = 16kHz 고정 강제**: 어댑터 진입부에서 입력 샘플레이트 != 16000이면 `librosa.resample`로 리샘플. v1의 transcript 어긋남 사고 재발 방지.
- **Pipecat 통합**: `WhisperSTTService` 직접 사용하거나 우리 래퍼 어댑터(ADR-012) 경유
- **언어 = ko 고정** (한국어 전용 — 영어 입력 시도는 P1으로 분리)
- **VAD 필터 = 활성**: Whisper 자체 VAD + silero-vad 이중 적용 (ADR-007)

### config 예시

```yaml
stt:
  model: "large-v3-turbo"
  device: "cuda"
  compute_type: "float16"
  language: "ko"
  beam_size: 1            # 지연 우선 (5는 정확도 우선)
  vad_filter: true
  resample_to: 16000      # 항상 16kHz로 강제
```

## 결과

### 긍정
- v1 검증된 모델 그대로 유지 — 회귀 위험 0
- 16kHz 강제로 v1 known_constraint 해소
- Pipecat 공식 어댑터 사용 가능 — 자체 어댑터 코드 최소화

### 부정
- 한국어 단일 — 영어 코드 스위칭 발화 시 정확도 저하 가능 (P1 검토)
- `large-v3-turbo` ~1.5GB VRAM — 모델 동시 상주 시 GPU 메모리 예산 검토 필요(ADR-015)

## 대안

| 후보 | 탈락 사유 |
|---|---|
| whisper.cpp | GPU 활용 약함, Python 통합 부담 |
| OpenAI Whisper 원본 | faster-whisper(CTranslate2) 대비 3~5배 느림 |
| AssemblyAI / Deepgram | 클라우드 의존 — ADR-002 로컬-only 위반 |
| MMS / NeMo Korean | 한국어 단일 데이터셋 의존, large-v3-turbo 대비 우위 없음 |
| faster-whisper distil-large-v3 | 한국어 정확도 검증 불충분, 영어 위주 |
