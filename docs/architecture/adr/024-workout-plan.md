# ADR-024: 운동 플랜 — 주간 목표 + 조정은 확인

- **상태**: Proposed (2026-06-10)
- **관련 ADR**: ADR-013 (능동 코치/확답), ADR-023 (컨디션), ADR-025 (메모리/기준선), ADR-028 (첫 체력검증)

## 컨텍스트

v3 는 단발 `propose_set`(고정값 제안)에 그쳤다. v4 는 목표 기반 플랜으로 격상해, 컨디션(ADR-023)에 따라 강도를 조절하는 **지속 코칭**을 지향한다.

핵심 제약은 회고가 강하게 경고한 패턴 — **"LLM 이 사용자 확답 없이 운동/계획을 마음대로 바꾸는 것"**(v3 ConfirmRule 로 막은 버그). 컨디션 기반 조절이 의미를 가지려면 *기준선/목표*가 있어야 한다(일일 자유로는 "줄일까요?"가 성립 안 함).

## 결정

- **주간 목표 설정**: 예 "이번 주 푸시업 3회" → 백엔드가 일자별로 분배.
- **조정은 확인**: 컨디션·실패에 따른 변경은 **코치가 제안 → 사용자 동의** 후 적용. **자동 변경 금지**(회고 ConfirmRule 정신).
- **기준선**: 첫 체력검증(ADR-028) 결과 + 메모리(ADR-025 1층 `fitness_baseline`). 부상/제약(ADR-025)은 안전 가드로 항상 반영.
- **tool-use**: LLM 이 plan/db 를 도구로 호출해 제안을 생성(ADR-013 instructor 확장). 실행(카운팅 시작)은 기존 확답 게이트 유지.
- **구현 순서**(점진): v4 초기 = 주간 목표 세팅 + 일자 분배. 컨디션 기반 자동 제안은 컨디션 레이어(ADR-023) 안정 후 얹는다.
- **안 행복한 경로**:
  - 목표 미설정 → 일일 자유 모드로 fallback(그날 컨디션 기반 단발 제안).
  - 목표 실패 누적 → 코치가 "목표를 낮출까요?" 제안(확인).

## 결과

### 긍정
- 컨디션 기반 조절의 기준선 확보 — 비전 차별점 성립
- 자동 변경 배제로 신뢰·안전(회고 원칙 준수)
- 진척 추적·동기부여 가능

### 부정
- 확인 단계로 약간의 마찰
- 플랜 상태(목표·진척·조정 이력) 관리 복잡도 증가
- tool-use 정확도에 의존(ADR-029 모델 검증 필요)

## 대안

| 후보 | 탈락 사유 |
|---|---|
| 주간 목표 + 자동 조정 | 사용자 모르게 플랜 변경 — 회고가 가장 강하게 경고한 패턴 |
| 일일 자유(목표 없음) | 컨디션 조절의 기준선 부재 — 비전과 불일치 |
