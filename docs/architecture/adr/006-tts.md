# ADR-006: TTS 어댑터 — Qwen3-TTS (Base + 참조 음성)

- **상태**: Accepted (Confirmed 2026-05-29, Phase 6)
- **이력**:
  - 초기: Kokoro 단독
  - 1차 개정: Kokoro + Qwen3-TTS 교체형 (2026-05-21)
  - 2차 개정: Kokoro 제거, Qwen3-TTS 단독 (2026-05-28) — 이유 아래
  - Phase 6 최종 확정: `config.tts.active = "qwen3"` (2026-05-29) — 아래 "Phase 6 최종 결정"
- **관련 ADR**: ADR-004 (LLM 듀얼, 동일 철학), ADR-010 (어댑터 인터페이스)

## 컨텍스트

1차 개정 당시 Kokoro-82M을 한국어 비교 후보로 포함했으나, Phase 2C 구현 조사 결과 **Kokoro 0.9.4(최신)는 한국어를 지원하지 않음**이 확인되었다 (지원 언어: 영/스/불/힌/이/일/포/중 9종, GitHub issue #294). 대체 후보인 Fish Speech, GPT-SoVITS는 모두 reference audio가 필수여서 Qwen3-TTS와 차별점이 없다.

따라서 **Qwen3-TTS Base 모델 단독**으로 결정하며, 향후 청취 비교가 필요할 경우 Fish Speech / GPT-SoVITS를 추가 어댑터로 구현한다.

## 결정

### 어댑터 구조

```
app/adapters/tts/
├── protocol.py      # TTSAdapter Protocol (ADR-010)
├── qwen3.py         # Qwen3TTSAdapter
└── __init__.py      # dict 매핑 (get_tts_adapter)
```

### 모델

- **Qwen3-TTS-12Hz-1.7B-Base** (`Qwen/Qwen3-TTS-12Hz-1.7B-Base`)
- 한국어 명시 지원, Apache 2.0
- 참조 음성(ref_audio) 방식 — 3초 이상 한국어 WAV 1회 준비 후 고정 사용
- VRAM: ~4GB (bfloat16 기준), RTX 5090에서 여유 있게 구동

### 참조 음성 설정

```yaml
tts:
  active: "qwen3"
  qwen3:
    model_id: "Qwen/Qwen3-TTS-12Hz-1.7B-Base"
    ref_audio_path: "data/ref_voice.wav"   # 3초+ 한국어 WAV
    ref_text: ""                           # 선택적 전사 텍스트
    voice: "default"
    speed: "1.0"
```

### 출력

- WAV bytes (16-bit PCM mono, 12 kHz)
- `synthesize()` → 전체 WAV, `stream()` → 단일 청크 (MVP, P1에서 개선)

### pip 패키지

```
qwen-tts>=0.1.1
torch>=2.4
```

## 결과

### 긍정
- 한국어 품질 확인된 유일한 로컬 TTS 실용 선택지
- Apache 2.0 라이선스
- 참조 음성 1회 설정 후 고정 → 피트니스 코치 음성 일관성 확보
- 향후 Fish Speech / GPT-SoVITS 추가가 동일 구조로 가능

### 부정
- 참조 음성 파일 준비 필요 (초기 설정 비용)
- `stream()` 미구현 (MVP에선 전체 합성 후 단일 청크)
- Qwen3-TTS 0.1.1 출시 4개월 — 운영 중 버그 가능성

## 대안

| 후보 | 탈락 사유 |
|---|---|
| Kokoro-82M | 한국어 미지원 확인 (v0.9.4, 2026-05-28) |
| Fish Speech (fishaudio/openaudio-s1-mini) | ref audio 필수, 12–24GB VRAM |
| GPT-SoVITS | ref audio 필수, 복잡한 모델 구성 |
| Qwen3-TTS CustomVoice | 프리셋 화자만 가능 — 코치 음성 고정 불가 |
| edge-tts (Microsoft) | 네트워크 의존 (ADR-002 로컬 우선 위반) |

## Phase 6 최종 결정 (2026-05-29)

PRD 부록 D의 "Kokoro vs Qwen3-TTS 한국어 30문장 청취 → `config.tts.active` 확정" 항목 처리:

- **청취 A/B 비교는 무의미(moot)**: Kokoro 0.9.4가 한국어를 지원하지 않아 2차 개정에서 이미 제외됨. 한국어 가능 로컬 TTS 후보는 Qwen3-TTS 단독이므로 비교 대상이 존재하지 않는다.
- **확정**: `config.tts.active = "qwen3"` (Qwen3-TTS-12Hz-1.7B-Base). RTX 5090에서 합성 정상 동작 확인(GPU 스택 검증, 2026-05-29).
- **잔여(사용자 수행)**: 주관적 음질·참조 음성(ref_audio) 적합성 청취는 사람의 판단이 필요하며 에이전트가 대신 통과시킬 수 없다. 현재 `data/ref_voice.wav`는 게임 캐릭터 보이스(상업/배포 금지 표기, `config.yaml` 주석 참조)로 **개인 로컬 실험용**이다. 외부 배포 전환 시 라이선스 문제 없는 참조 음성(예: AI Hub 음성)으로 교체할 것 — 코드 변경 없이 WAV 파일 + `ref_text`만 교체하면 됨.
- **향후**: 음질 불만족 시 동일 어댑터 구조로 Fish Speech / GPT-SoVITS를 추가 후보로 구현 가능(둘 다 ref audio 필수).
