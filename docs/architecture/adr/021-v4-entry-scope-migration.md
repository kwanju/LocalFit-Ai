# ADR-021: v4 진입 — 비전·스코프·마이그레이션 + UX 3모드

- **상태**: Proposed (2026-06-10)
- **관련 ADR**: ADR-002 (단일 사용자) 계승, v4 시리즈(022~031) 우산
- **근거 문서**: `docs/future/v4-vision.md` §0-2 (deep interview 결과)

## 컨텍스트

v3-rewrite 가 master 에 동결됐다(2026-06-09). v4 는 단발 응답 코치를 넘어 **지속성 있는 헬스 코치**로 확장한다. 방향은 deep interview(2026-06-10)로 확정했고, 본 ADR 은 v4 시리즈 전체의 스코프·전제를 고정하는 우산 ADR 이다.

## 결정

### 비전·스코프
- **v4 = 운동 + 컨디션/건강 코치.** 컨디션(수면·피로·근육통) 트래킹 + 컨디션 기반 강도 조절을 핵심으로 추가.
- **제외**: 영양 추적, 일과 비서(범용 스케줄 관리). 스코프 과대 방지(YAGNI).

### 사용자 전제
- **단일 사용자 유지** (ADR-002 계승). 멀티유저(한 인스턴스 다수 사용자)·인증·권한 코드 금지.
- 단, **다중 하드웨어 배포**("친구도 자기 머신에 설치")는 고려 — config 모델 교체(ADR-029)로 흡수, 분기 코드 없음.
- **로컬-only(ADR-002) 부분 완화**: Google Calendar 연동(ADR-022)에 한해 외부 API 허용 — 본인 계정 1개·캘린더 데이터만. 운동기록·메모리·음성은 로컬 유지. (우산 ADR 로서 이 예외를 명시 — ADR-002 본문은 frozen 이므로 여기서 조정.)

### 마이그레이션
- **v4 새 브랜치 `v4-rewrite`**, v3 master 동결 유지.
- v4 ADR 이 v3 ADR 과 충돌하면 **v4 우선**. 충돌 ADR 은 새 번호로 발행 + 기존 Status 를 `Superseded by`로 갱신.

### UX — 4모드 → 3모드
- v3 의 4모드(C2C/C2S/S2S/S2C) 중 **S2C(음성입력→텍스트출력) 제거 → 3모드(C2C/C2S/S2S)**.
- 사유: 음성으로 말하고 답을 텍스트로만 받는 사용 맥락이 어정쩡(손이 자유로우면 S2S, 조용해야 하면 C2C/C2S 가 더 자연스러움).

### 지속성 모델
- v3 **세션 모델 유지** + **백그라운드 스케줄러** 추가(운동 시간/체크인 리마인드). 상시 가동·선제 발화 에이전트는 아님.

## 결과

### 긍정
- 스코프 명확(운동+컨디션) — 회고의 "안 행복한 경로/YAGNI" 원칙 준수
- master 동결로 v3 작동본 보존하며 v4 점진 개발
- 3모드로 UX 단순화

### 부정
- 별도 브랜치 운영 부담(머지 전략 필요)
- 컨디션/스케줄러/메모리 추가로 v3 대비 상태 복잡도 증가
