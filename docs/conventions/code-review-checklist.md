# 코드 리뷰 체크리스트

매 에이전트 작업 완료 시 사용. 5분 내 통과해야 함. 통과 못 하면 에이전트에게 수정 요청.

## 체크리스트 사용 방법

1. 에이전트가 작업을 완료하면 변경된 파일 목록을 받는다
2. 아래 체크리스트를 순서대로 확인한다
3. 한 항목이라도 ❌면 에이전트에게 구체적으로 수정 지시
4. 모두 ✅면 커밋

---

## A. ADR 준수 (가장 중요, 30초)

- [ ] 새 의존성 추가가 있다면 ADR-015 패키지 매니저(uv/pnpm)로 추가되었는가
- [ ] 외부 라이브러리 호출이 ADR-010 어댑터를 경유하는가 (LLM, STT, TTS)
- [ ] DB 접근이 ADR-007 Repository 패턴을 경유하는가 (단순 SELECT는 예외)
- [ ] 멀티유저 코드(`user_id` 파라미터, 인증 등)가 없는가 (ADR-002 단일 사용자)
- [ ] 추상화 레이어를 임의로 추가하지 않았는가 (YAGNI)
- [ ] 폴더 구조가 설계 문서대로인가 (`app/core/`, `app/adapters/` 등)

## B. PRD 준수 (1분)

- [ ] PRD에 없는 기능을 임의로 추가하지 않았는가
- [ ] 부상 키워드 처리가 LLM 우회 규칙 기반인가 (PRD 6장)
- [ ] 카운팅이 LLM 응답과 독립적으로 동작하는가 (PRD 4-1)
- [ ] 한국어 메시지가 사용자 대면 코드에 사용되었는가
- [ ] PRD 미결 사항을 임의 결정하지 않았는가 (결정 시 ADR 추가 필요)

## C. 코딩 컨벤션 (1분)

- [ ] 모든 함수에 타입 힌트가 있는가
- [ ] 함수가 50줄 이내인가 (예외 시 사유 명시)
- [ ] 매직 넘버가 없는가 (config 또는 상수)
- [ ] f-string 사용 (% 포매팅 금지)
- [ ] 한 파일에 한 책임만 있는가
- [ ] 임포트 순서 (표준 → 서드파티 → 로컬)

## D. 에러 처리 (30초)

- [ ] try-except가 빈 pass로 끝나지 않는가
- [ ] 예외 발생 시 `logger.error` 또는 `logger.exception` 호출
- [ ] 사용자 대면 에러 메시지가 한국어인가
- [ ] 시스템 로그가 영어인가 (디버깅 편의)
- [ ] 외부 호출(LLM, STT, TTS, DB)에 타임아웃이 있는가

## E. 테스트 (1분)

- [ ] 새 함수에 단위 테스트가 추가되었는가 (어댑터, core 모듈)
- [ ] 테스트가 실제로 동작하는가 (`pytest` 실행 확인)
- [ ] 외부 의존(GPU, Ollama)이 있는 테스트는 마킹되었는가 (`@pytest.mark.gpu` 등)
- [ ] 한국어 테스트 데이터가 포함되었는가 (해당 시)
- [ ] Mock이 ADR-010 Protocol을 준수하는가

## F. 비동기·동시성 (30초)

- [ ] FastAPI 엔드포인트가 `async def`인가
- [ ] 동기 블로킹 코드(file I/O, 네트워크)가 `asyncio.to_thread`로 감싸졌는가
- [ ] 카운팅 박자 스케줄러가 `time.monotonic()` 기반인가 (sleep 누적 금지)
- [ ] LLM/TTS 작업이 cancelable Task인가 (인터럽트 대응)

## G. 보안·프라이버시 (30초)

- [ ] 시크릿이 코드에 하드코딩되지 않았는가 (config.yaml, .env 사용)
- [ ] 외부 통신이 `127.0.0.1`로 제한되었는가 (P0 ADR-002)
- [ ] 사용자 입력이 SQL/LLM 프롬프트에 안전하게 삽입되는가
- [ ] 로그에 개인정보 평문 노출이 없는가

## H. 모델·프롬프트 (1분)

- [ ] 모델 이름이 하드코딩되지 않았는가 (config.yaml 참조)
- [ ] 안전 규칙이 시스템 프롬프트 최상단에 있는가 (PRD 6장, 부록 B-2)
- [ ] `keep_alive` 파라미터가 의도된 값인가
- [ ] 프롬프트 템플릿이 `app/prompts/`에 분리되었는가

## 자가 점검 결과 양식 (에이전트에 요구할 형식)

매 작업 완료 시 에이전트가 다음 형식으로 보고하게 한다:

```markdown
## 작업 완료 보고

### 변경 파일
- `app/adapters/llm/ollama.py` (신규)
- `tests/unit/test_llm_ollama.py` (신규)

### ADR 준수
- ADR-003 (Ollama): ✅ 사용
- ADR-010 (어댑터 Protocol): ✅ 시그니처 일치 확인
- ADR-011 (keep_alive): ✅ 동적 override 지원

### 테스트 실행
- `pytest tests/unit/test_llm_ollama.py`: 5 passed
- CLI smoke test: `python -m app.adapters.llm.ollama "안녕"` → 응답 확인

### 사용자 확인 필요 사항
- Qwen 3.5 9B 모델이 `ollama list`에 보이는지 확인 부탁
- (없으면 `ollama pull qwen3.5:9b` 실행 후 재시도)

### 자가 체크리스트
- A. ADR 준수: ✅
- B. PRD 준수: ✅
- C. 컨벤션: ✅
- D. 에러 처리: ✅
- E. 테스트: ✅
- F. 비동기: ✅
- G. 보안: ✅
- H. 모델/프롬프트: N/A (이 작업 범위 외)
```

이 보고를 받으면 사람이 위 체크리스트로 30초 cross-check.

## 위반 시 처리

### 사소한 위반 (컨벤션, 누락된 로그)
에이전트에게 구체적 항목 지적하며 수정 요청.
예: "C-2 함수가 50줄을 초과합니다. `_validate_input` 함수로 추출해 주세요."

### 중대한 위반 (ADR 무시, 임의 추상화)
변경 사항 전체 revert 후 다시 시작.
ADR을 명시적으로 참조하며 재요청.

### 반복적 위반
AGENTS.md에 해당 규칙을 더 강조하여 추가.
체크리스트의 우선순위 조정.
